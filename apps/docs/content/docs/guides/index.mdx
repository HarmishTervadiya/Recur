# Guides

Structured, end-to-end guides for merchants integrating Recur. Each guide walks you through a complete workflow from start to finish.

## Guides

| Guide | What you'll learn |
|-------|-------------------|
| [Getting Started](/docs/guides/getting-started) | Prerequisites, required vs optional steps, merchant & subscriber setup paths |
| [Backend Integration](/docs/guides/backend-integration) | Full server-side flow: SDK setup, create apps/plans, manage subscriptions, handle payments |
| [Webhooks](/docs/guides/webhooks) | Register endpoints, event types, payload format, signature verification, retry handling |
| [API Keys](/docs/guides/api-keys) | Create keys, authenticate requests, scopes, rotation, and security best practices |

## Which guide do I need?

- **I'm a merchant who wants to accept recurring payments** → Start with [Getting Started](/docs/guides/getting-started), then [Backend Integration](/docs/guides/backend-integration)
- **I need real-time payment notifications** → [Webhooks](/docs/guides/webhooks)
- **I'm building a server-to-server integration** → [API Keys](/docs/guides/api-keys)
- **I want SDK reference docs** → See the [SDK section](/docs/sdk/quickstart)
