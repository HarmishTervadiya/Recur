The Problem

Crypto recurring payments are broken

GoCardless literally says "Cryptocurrency systems don't inherently lend themselves to recurring billing." We fixed that.

Manual signing every cycle

Users must approve every single payment. Miss one? Subscription cancelled. Churn rate: catastrophic.

Recur fix: Sign once, keepers auto-collect

Uncapped token approvals

Smart contracts with uncapped withdrawal rights. Users give infinite access. One exploit = total loss.

Recur fix: Capped SPL delegate per cycle

No Solana SDK exists

Stripe is EVM-only. BoomFi is multi-chain generic. Nobody built native recurring billing for Solana. Until now.

Recur fix: 3-line React SDK, Solana native
How it works

Four steps. One transaction.

From approval to payment in 400ms. No intermediaries. No redirect URLs. Pure Solana.

01

SPL Approve

User signs one transaction. SPL delegate approval with capped amount per billing cycle.

02

PDA Created

Subscription account stored on-chain via Program Derived Address. Immutable and transparent.

03

Keeper Fires

Automated keeper triggers payment every billing cycle. No user action needed. Ever.

04

Merchant Receives USDC

USDC lands in merchant wallet + webhook fires. No bank. No intermediary. 400ms.

Why Not Stripe?

Stripe validated the market.
We built it better.

Stripe entering stablecoin subscriptions proves this is real. Here's what they can't do.

Solana Native

Stripe is EVM-only (Polygon, Base). No Solana. Recur: 400ms finality, $0.001 gas, no bridge complexity.

Capped Approvals

Stripe's Bridge contracts allow uncapped withdrawals. Recur: SPL delegate with per-cycle caps. Users see exactly what they approve.

Any Country, No KYC

Stripe requires US-registered merchants. Recur: 195+ countries, wallet-to-wallet, no bank account needed.

Developer-First SDK

Stripe: checkout sessions, redirect URLs, webhook endpoints. Recur: npm install @recur/react, done.
