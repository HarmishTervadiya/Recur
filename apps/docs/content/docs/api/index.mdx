# API Reference

The Recur REST API serves two audiences:

- **Merchants** — manage apps, plans, subscriptions, and view payment history
- **Subscribers** — view and manage their subscriptions

Base URL: `http://localhost:3001` (development) or your deployed API URL.

- [Authentication](/docs/api/authentication) — wallet-based nonce/signature auth flow
- [Merchant API](/docs/api/merchant) — apps, plans, webhooks, API keys
- [Subscriber API](/docs/api/subscriber) — subscription management
