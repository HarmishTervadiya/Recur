Design identity
Direction in one sentence
"Stripe discipline meets Solana dark energy — brutally clear, technically beautiful, zero fluff."
Dark-first. Purple-led. Code-forward. Every section should feel like a developer tool that happens to look incredible. Think Linear + Vercel + Superfluid: the confidence of enterprise SaaS with the aesthetic of a Web3 protocol. Nothing decorative lives here — everything visual earns its place by making the product more understandable.
Competitor analysis
What to steal, what to avoid
Stripe billing docs
The "how it works" numbered step flow
Stripe never uses flashy graphics to explain subscription flow — they use clean numbered steps with a product screenshot on the right. Each step is one sentence. Copy this exact structure for your "how Recur works" section. It reads as authoritative, not salesy.
BoomFi subscriptions page
The "how it works" horizontal card journey
BoomFi uses 4 horizontal cards — Authorise, Confirm, Notify, Charge — each with an icon and one line of text. Clean and scannable. Steal this layout but make yours darker, more technical, and replace their generic icons with Solana-specific language (SPL Approve, Keeper, PDA, Webhook).
Loop Crypto payments
The embedded pay widget demo
Loop embeds a live-looking payment component in the landing page itself. It's interactive and converts well because it makes the product tangible. Do the same — embed a fake but pixel-perfect <RecurButton /> that can be clicked to show the Phantom modal simulation.
Sablier
The stream visualisation animation
Sablier uses an animated bar that fills over time to represent streaming money. Adapt this: show an animated "next billing" countdown bar with a USDC pulse when payment fires. Turns an abstract concept into something visceral.
Superfluid (Cuberto redesign)
Dark background, vivid purple accent, large type
Superfluid's redesign by Cuberto nailed the aesthetic: near-black background (#08080F), vivid purple as the only accent, oversized bold headline, clean sans body text. This is the exact visual register Recur should hit. Not neon — sophisticated. Purple is the common thread between Recur and Solana's brand identity.
GoCardless
The pricing transparency section
GoCardless has a brutally honest "here's what you pay vs competitors" section with a table. Adopt this exact format but make the numbers devastating: Stripe at $0.59, Recur at $0.075. A table that makes competitors look expensive is more powerful than any copywriter.
Mercuryo
The "trusted by" logo strip with volume stat
Mercuryo leads with "$5B+ processed" and a logo row. Pre-launch, adapt this: list Superteam, Colosseum, Devnet integrations — and put "built at Frontier Hackathon 2026" as a trust signal. Credibility before social proof is available.
INQUD recurring payments — AVOID THIS
Wall of copy, text-heavy hero
INQUD's recurring payments page opens with a paragraph. Never do this. The hero should have a max 12-word headline, 2-line subhead, and two CTAs. Everything else lives in sections below the fold. Developers scan, they don't read.

Page structure
Exact section order — give this to your dev
01
Nav — sticky, minimal
Logo left, 3 text links center, "launch app" button right. Transparent background that transitions to #08080F on scroll. No hamburger on desktop.
02
Hero — the only section that matters
Badge pill ("built on Solana") → H1 max 12 words → 2-line subhead → 2 CTAs → 4 stat chips. No hero image — a live animated RecurButton demo or a subtle grid pattern is enough. Full viewport height.
03
Problem — 3 pain points in 3 cards
Manual signing / Capital lockup / No SDK. Dark cards with a one-sentence description each. Optional: crossed-out competitor approach on each card, Recur's fix below.
04
How it works — 4 step horizontal flow
User approves SPL delegate → PDA created → Keeper fires every 30 days → Merchant gets USDC + webhook. Steal BoomFi's horizontal card pattern. Add a faint connector line between cards. On mobile: vertical stack.
05
SDK code block — developer hook
Left: 3 steps (install, import, render). Right: dark terminal-style preview. Copy button. Tab switcher for npm / yarn / pnpm. This is where developer trust is built.
06
Pricing — transparent, devastating
$0.05 flat + 0.25% volume + $49/mo Pro. Table comparing to Stripe (2.9% + $0.30) and GoCardless ($0.20 min). Pro tier card elevated with a purple border.
07
Vs Stripe — the credibility section
Stripe validated the market. Here's what they can't do: 4 pills (Solana only / any country / USDC direct / game SDK). One sentence per point.
08
CTA footer — one job
"Start building in 5 minutes" → "read the docs" link + "connect wallet" primary button. Dark background, centered, generous vertical padding.
Motion & animation
What to animate — and what to leave still
Keeper pulse: a green dot that pulses every 60 seconds in the hero — "Keeper is live." Subtle. Never distracting. Think the Vercel deployment indicator.
Transaction stream: in the dashboard mockup section, payments should fade in from the top at a 2–3 second interval. Shows Recur is working, live.
Code highlight: on scroll into the SDK section, the code block types itself in with a cursor. 800ms total. Stop once visible — don't loop.
Stat counters: the 4 hero stat chips count up from 0 on page load. $0.001 counts from 0 in 600ms. Use CSS counter-animation or JS IntersectionObserver.
Section fade-in: every section fades in with a slight upward translate (translateY 12px → 0) as it enters viewport. 300ms ease-out. Standard, professional, expected in 2026.
Button hover: primary CTA scales to 1.02 and background lightens by 10%. Not a colour shift — a brightness lift. Keeps the purple consistent.
What NOT to animate: the hero background (no particle fields, no floating orbs), card borders (no "glow on hover"), logos in the trust strip. Stillness = confidence.
Component rules
The 6 rules your dev must not break
One purple, one white — that's it
Every CTA is either #7C3AED filled (primary) or text-only #A78BFA (secondary). No green buttons, no blue buttons, no orange. Purple is the brand signal — diluting it with other CTA colours kills identity.
Borders not shadows
Cards use 0.5px border #2A2A3E on dark backgrounds. Never box-shadow for depth — it reads as dated on dark UIs. Depth comes from layered background colours (#08080F → #0D0D14 → #12121C).
Mono font for all on-chain data
Wallet addresses, tx hashes, amounts, timestamps — always JetBrains Mono or Fira Code. This signals "this is real chain data." Never render a Solana address in Inter — it looks like a typo.
Green is success only
#34D399 is reserved for confirmed payments, active status, positive deltas. Never use it decoratively. Users are conditioned to read green as "money moved." Use it that way and only that way.
Max container width 1120px
Content never bleeds wider than 1120px. Padding left/right min 24px on mobile. Sections are centred. Full-bleed backgrounds are allowed (hero, CTA footer) but content inside stays in the container.
Mobile-first, not mobile-adapted
The code block, how-it-works cards, and pricing table — all built mobile-first and scale up. Solana game devs check landing pages on phone. If it sucks on iPhone 15, it doesn't ship.
