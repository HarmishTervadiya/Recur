# Integration

Recur ships three layers so you can integrate at exactly the level of control you want — no Web3 knowledge required.

| Layer | Package | Use when |
|---|---|---|
| **Components** | `@recur/react` | You want a button or modal that just works |
| **Hooks** | `@recur/react` | You want to keep your own UI but skip the wallet/transaction plumbing |
| **Server** | `@recur/server` | You want to verify webhooks in your backend |
| Advanced | `@recur/sdk` | You want raw transaction builders and full control |

## Three-step setup

### 1. Install

```bash
bun add @recur/react @recur/server
```

### 2. Wrap your app

```tsx title="app/layout.tsx"
"use client";
import { RecurProvider } from "@recur/react";

export default function RootLayout({ children }) {
  return (
    <RecurProvider
      apiBaseUrl={process.env.NEXT_PUBLIC_RECUR_API_URL!}
      cluster="devnet"
    >
      {children}
    </RecurProvider>
  );
}
```

### 3. Drop in a button

```tsx
import { SubscribeButton } from "@recur/react";

<SubscribeButton appId="app_xxx" planId="plan_xxx">
  Subscribe — $9.99/month
</SubscribeButton>
```

That's it. The button handles wallet connection, sign-in, USDC delegation, and registration with the Recur API. On success the merchant's webhook receives `subscription_created`.
